HouseofOrange

题目给出了三个函数，漏洞点在 Edit 函数可以超大堆溢出。给了三次 Add 机会，三次 Edit 机会。分步骤学习一下。

思路

先搭好框架：

1.堆溢出改小 Top chunk 的 size ，利用 sysmalloc 把 Top chunk 🐑 到 unsortedbin 里面。

这里的 0xF81 是我们伪造的 Top chunk 的 size，我们可以通过动调保留低三位，也可以自己伪造，具体得满足以下要求：

  • 伪造的 size 必须要对齐到内存页
  • size 要大于 MINSIZE(0x10)
  • size 要小于之后申请的 chunk size + MINSIZE(0x10)
  • size 的 prev inuse 位必须为 1
  1. ADD 会从被扬的 Old Top 里面分割出一个 unsortedbin 的堆块，Show 泄露 libc 和堆地址。

这一部分比较好理解，需要注意的是 Add(0x400,'c') 不能取小了，否则拿不到堆地址。后面 Edit(0x10,'c'*0x10) 的原因是 printf 函数存在 \x00 截断，我们泄露完 libc 就要填充一下来泄露堆地址。

3.重点是利用 Unsortedbin Attack 打 _IO_list_all 为 main_arena+88 ，再次 Add 时，FSOP getshell。

这里巧妙的是，我们构造 fake_file 的同时把 chunk 的 size 改成了 0x60，保证了之后会被扬到 0x60 的 smallbins 里面。

我们然后我们使用 Unsortedbin Attack 在 _IO_list_all 的地方写上了 main_arena + 88 ，FSOP 的时候就会遍历 _IO_list_all 根据 _chain 找到 main_arena + 88 + 104 = main_arena + 192 位置我们扬进去的 fake_file。
