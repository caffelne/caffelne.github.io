Pwn刷题笔记

发表于2022-03-12|更新于2022-03-16|CTFPwn堆栈
|阅读量:

Practice makes perfect.

​ 记录一下做题时候的一些疑问和后面自己的一些思考，所以 exp 相当的 💩 ，然后以后自己过来复习的时候 exp 可能也看不懂，但思路和踩过的坑应该还是可以捡起来的。

Buuoj

0ctf_2017_babyheap

这题比较简单，琢磨琢磨给干出来了。

考点

  1. Fill 函数存在堆溢出，操作空间很大，可以打堆重叠后用 Dump 函数泄露信息
  2. Unsortedbin 泄露 main_arena 地址，泄露 libc
  3. Fastbin 构造错位，任意分配堆块到 __malloc_hook
  4. 从一万种可能的 libc 里面选一个对的来进行一个 onegadget 的写

Exp

思考

堆溢出

  1. 构造数据的时候如果要改 chunk 3 必须保证 size 大小正确。
  2. calloc 分配堆块的时候会清空对应内存

Fastbin 任意分配堆块

​ 这里因为要找到确定的 size 来通过对应检测，选择了 x64 下一般地址中不会变的 0x7f ，然后 0x7f 是属于 calloc(0x60) 这个大小的 Fastbin 范围，所以选用的都是 add(0x60) 。

​ 具体偏移（上面的什么 0x1b 0xb+8）算起来费脑子，建议直接动态调试，毕竟偏移这玩意不会变，可以省好多脑细胞。

——2022/3/12 21:36

N1_note

考点

  1. Unsortedbin 泄露 libc
  2. 存在 UAF 时构造 Tcache Poisoning 任意分配堆块
  3. PIE 开启时的 getshell

Exp

思考

PIE

开启了 PIE 后想打 got 表较为困难，应该首先考虑写 hook。

Tcache Poisoning

很好用的攻击技巧，但要保证指向合法地址，比如这题开了 PIE 明显不能直接写 elf.got[‘free’] 到 next 指针。

——2022/3/14 1:36

vn_pwn_easyTHeap

这题最后进行到写 onegadget 的时候卡住了，试过了下面的 onegadget 都不行。就这么不管了挺可惜的，过程中练习了 Tcache 的相关 attack ，还是记录一下。

考点

  1. 次数限制的 add 和 delete，free 后可以 show 泄露信息但不能直接 edit 时候的堆利用
  2. 无检查的 Tcache double free
  3. 劫持 tcache_perthread_struct
  4. 写 one_gadget

Exp

思考

劫持 tcache_perthread_struct

  1. Tcache poisoning 配合 Tcache double free: 利用 Tcache double free 修改 fd 指针(不含 chunk 头，指向 user 部分)分配到 tcache_perthread_struct (利用 UAF ，通过 show 函数泄露堆地址，减去相应偏移后得到地址)。
  2. 分配到 struct 后，利用 edit 函数把 counts 数组全部写成大数字，那么此时就不会往 Tcache 里面塞堆块了， struct 就会被扔到 unsortedbin 里面。
  3. 而此时的 unsortedbin 里面只有 struct 一个 chunk，那么我们分配一个小 chunk 时就会从其中分割，就可以利用分割下来的前半部分 chunk，从而能通过 edit 函数往 entries 数组写 __malloc_hook 地址。
  4. 而后再申请对应大小堆块时就会分配到 __malloc_hook ，从而可以利用 edit 函数写 onegadget 地址 getshell(失败了XD)。

——2022/3/15 23:59

CTFshow

吃鸡杯 - ezheap

考点

  1. Edit 函数 Off by one 构造堆块重叠后通过 edit 和 show 函数达到任意读任意写的效果。
  2. 泄露堆地址后，计算出 Unsortedbin 的偏移泄露 libc
  3. 直接打 atoi 的 got 表来 getshell

Exp

思考
